### 模板 9：性能优化与调试
目标
对无限防守模式+ 肉鸽游戏进行性能分析和优化，确保 60FPS 流畅运行，减少 GC 分配。

上下文
使用 Unity Profiler 和 Memory Profiler 定位性能瓶颈

高频更新对象：Player（每帧检测目标）、技能效果、敌人移动、enemy的动效

高频生成/销毁对象：子弹、特效飘字、技能特效

场景包含大量敌人时，Physics2D 检测的性能消耗较高

优化要求
对象池全面适配：确保所有动态生成的 GameObject 都通过对象池管理

Update 优化：

Player的目标检测频率可以降低到每 0.2 秒一次（使用 InvokeRepeating 或协程），而非每帧检测

非激活状态的enemy禁用 Update（通过 enabled 属性控制）

大量敌人时，使用对象池预加载技术

Physics2D 优化：

使用 LayerMask 排除不必要的碰撞检测

Player的射程检测改用 OverlapCircleNonAlloc 避免每帧分配新数组

UI 优化：

界面之间切换时避免频繁的 SetActive，改为 CanvasGroup alpha 控制或使用 Canvas.enabled

TextMeshPro 文本更新尽量合并，减少重建频率

代码优化：

避免在 Update 中调用 GetComponent

缓存组件引用到私有字段

减少 foreach 循环中不必要的装箱操作

输出要求
提供性能分析报告模板（Profiler 截图分析要点）

提供关键优化函数的重构代码（如 Tower 的目标检测频率控制）

提供优化前后的对比测试方法

提供移动端特定的性能配置建议
