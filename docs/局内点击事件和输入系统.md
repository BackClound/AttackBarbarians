### 模板 2：配置项目 settings 和输入系统
目标
为 Unity 2D 无限防守肉鸽游戏配置项目设置和新的 Input System。

上下文
使用 Unity Input System Package 1.7.0+（通过 Package Manager 安装）

游戏类型：2D 无限防守 + 肉鸽随机升级系统（主要交互是通过获得buff提升玩家技能及属性等级、暂停、选择肉鸽升级，设置音效，返回主页）

支持触屏

输出要求
生成 Input Action Asset（保存路径：Assets/Settings/Input/GameInput.inputactions），包含以下 Action Maps：

PlayerInput：暂停按钮、 buff按钮、设置按钮及音效设置按钮，返回按钮

UI：导航（Back）、暂停（Push）、buff UI、设置（Setting）、音效设置UI

生成对应的 InputHandler 脚本（命名为 PlayerInputHandler.cs），监听 Input Actions 并发布到 GameEvents

配置项目播放设置（Player Settings）：

支持的分辨率：1920x1080（全屏窗口可选）

默认图标和公司名称使用占位符

设置适当的质量设置，针对 2D 游戏优化

验证方式
在场景中挂载 PlayerInputHandler 组件，运行后检查 Console 是否可以捕获触摸位置、点击输入事件。
